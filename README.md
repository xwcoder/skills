# skills

这个仓库包含两个面向 Codex 的本地 skill，用来把用户给出的简短主题扩展成结构化的中文 Markdown 总结。

## skills

| Skill | 适用主题 | 主要用途 |
| --- | --- | --- |
| `topic-diver` | 通用学习或研究主题 | 拆解商业、历史、文化、科学、管理、产品、个人学习等非纯技术主题，并生成完整中文 Markdown 文档。 |
| `tech-topic-diver` | 技术学习主题 | 拆解编程语言、框架、基础设施、数据库、AI/ML、安全、架构、工程实践等技术主题，并生成完整中文技术文档。 |

## 使用方式

安装完成后，在对话中提出一个主题即可触发对应 skill。

通用主题示例：

```text
帮我系统总结一下「品牌定位」
```

技术主题示例：

```text
帮我系统总结一下「Kubernetes Ingress」
```

两个 skill 都会优先把主题扩展成覆盖面完整的 Markdown 文档，而不是只给学习路线。对于范围较大的主题，它们会先用少量选择题确认关注角度、深度和是否需要示例；如果用户要求快速输出，则会使用平衡默认设置直接生成文档。

## 输出特点

- 默认使用简体中文。
- 默认产物是适合保存为 `.md` 文件的 Markdown 内容。
- 会先梳理主题范围、核心概念、背景演进、关键维度、案例、对比、误区、实践建议和后续问题。
- 必要时会使用图示辅助说明，图片可以来自搜索或生成，例如全景图、架构图、时序图、数据流动图、生命周期图或对比图。
- `tech-topic-diver` 会更强调工程实践、内部机制、工具生态、选型权衡、常见坑和最佳实践。
- `topic-diver` 会更强调概念理解、社会/商业/文化等相关维度、现实案例、争议和分析框架。

## 维护说明

每个 skill 的完整行为定义都在 `skills/<skill-name>/SKILL.md` 中。
